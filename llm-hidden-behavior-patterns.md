# LLM 隐藏行为模式与触发方式

> LLM 对每个中文词都有一套"默认输出模板"。Prompt 工程的本质不是加约束，而是选模板。

> LLM 为什么有"默认输出模板"——它的基础认知机制（概率预测、统计模式匹配）详见 [LLM 基础认知与心智模型](llm-fundamentals.md)。

---

## 核心原理

LLM 被训练在海量文本上，每个词都关联着特定文体的输出分布。"总结"的训练数据是摘要（压缩），"汇总"的训练数据是报告（保真）。当你用某个词时，你激活的不是语义，而是**文体模板**。

```
加约束 = 试图覆盖默认模板（对抗模型，效果差）
换一个词 = 直接选对模板（顺从模型，效果好）
```

### 底层机制：高维关键词匹配

LLM 看起来远超关键词识别——它自然、灵活、能处理复杂语义。这种表象造成了一种反向误会："用关键词太 low 了，LLM 理解的是语义"。

但事实是：**LLM 从海量语料中学到了"关键词识别"的高维版本**。"总结"和"汇总"在语义上几乎同义，但 LLM 对它们的响应截然不同——因为训练数据中这两个词出现的上下文分布不同。LLM 没有"理解"这两个词的意思差异，它只是**匹配了不同的统计模式**。

```
人类视角：总结 ≈ 汇总（语义相近）
LLM 视角：总结 → [摘要语料分布]，汇总 → [报告语料分布]
          ↑ 两个完全不同的激活路径
```

这就是为什么换一个词比加一段约束更有效——你不是在和一个"理解语义的智能体"对话，你是在和一个"匹配统计模式的引擎"对话。找到那个统计模式恰好等于你想要结果的词，比用自然语言去"说服"它更可靠。

好的 prompt 不是加更多约束，而是找到那个让 LLM 的默认行为恰好等于你想要的结果的词。

## 一、信息密度控制

| 你想要的 | 别说 | 说 | 原理 |
|---------|------|-----|------|
| 保真不压缩 | 总结这段对话 | **汇总**这段对话 | 总结=压缩模式（丢弃"冗余"），汇总=整理模式（保真排列） |
| 要点不啰嗦 | 写一篇文章 | **列**出来 | 写=触发散文生成，列=触发结构化提取 |
| 精准不展开 | 解释一下 | **说**一下 | 解释=教科书模式（verbose+居高临下），说=直接给结论 |
| 要 concrete 不要 abstract | 学习这个项目的架构 | **抄**这个项目的架构 | 学习=抽象原则，抄=具体代码片段 |

---

## 二、行为方向控制

| 你想要的 | 别说 | 说 | 原理 |
|---------|------|-----|------|
| 精简不膨胀 | 优化这段代码 | **减**这段代码 | 优化=AI 默认加功能/加抽象，减=触发删减行为 |
| 只填缺口 | 改一下 | **补**一下 | 改=可能重写，补=只补缺失部分 |
| 要决策不要废话 | 帮我设计一个系统 | **定**一个方案 | 设计=过度工程化，定=做决策、选一个 |
| 果断判断 | 讨论一下这个方案 | **判**这个方案 | 讨论=两边都说、平衡，判=给结论、下判决 |

---

## 三、交互模式控制

| 你想要的 | 别说 | 说 | 原理 |
|---------|------|-----|------|
| 直接干不废话 | 帮我做一个 X | 做一个 X | "帮我"触发客套+确认流程，去掉触发直接执行 |
| 严格挑刺 | 评价这段代码 | **批**这段代码 | 评价=温和平衡，批= rigorous critique |
| 自信诊断 | 可能是 X 问题 | **是** X 问题 | "可能"触发不确定性/对冲，直接陈述触发自信判断 |
| 要方案不要清单 | 建议一个方案 | **选**一个方案 | 建议=列多个选项让你选，选=替你做决策 |

---

## 四、思维深度控制

| 你想要的 | 别说 | 说 | 原理 |
|---------|------|-----|------|
| 快速实用 | 思考一下 | **想**一下 | 思考=触发内部独白（长），想=触发快速实用判断 |
| 填充缺口 | 完善这个设计 | **补**这个设计 | 完善=添加新东西，补=只补缺失的 |
| 要减法 | 重构这段代码 | **砍**这段代码 | 重构=可能加抽象层，砍=只删不必要的 |

## 五、原理总结

### 5.1 为什么有效

LLM 的输出分布是训练数据的统计反映：

- "总结"出现的上下文 = 摘要、纲要、压缩文本 → 输出倾向压缩
- "汇总"出现的上下文 = 报告、清单、目录 → 输出倾向保全
- "优化"出现的上下文 = 增强、升级、改进 → 输出倾向加东西
- "减"出现的上下文 = 删减、精简、瘦身 → 输出倾向删东西

你选择的词决定了 LLM 从哪个文体分布中采样。

### 5.2 为什么约束不如换词

约束是**外力**——你告诉模型"不要做 X"，但模型的默认行为仍然是做 X，只是在做之前多了一层检查。这层检查会随着上下文变长、注意力分散而逐渐失效。

换词是**内因**——你改变了输入分布，模型从根源上就不会产出你不要的东西。不需要对抗，没有衰减。

### 5.3 通用方法论

当你发现需要加大量约束才能得到想要的结果时：

1. **停下来**：约束太多 = 你在对抗模型的默认行为
2. **找文体**：你想要的输出最像哪种文体？（报告？判决？清单？代码？）
3. **找触发词**：那种文体的标题/开头通常用什么词？
4. **换词测试**：用那个词替换当前的动词，看输出是否自然符合预期

> 最好的 prompt 是一个词，不是一段话。
