# 智力金字塔模型

**结论**：将系统智力组织为金字塔——底层消化海量数据，顶层只处理精英决策。每一层都是上一层的卸载目标，每一层过滤掉下一层不需要的信息。
**类比**：计算机多级存储（Cache → RAM → SSD → HDD）的梯度平衡，映射到 AI 工程的智力梯度。

---

## 金字塔结构

```
        ╱╲            L1  LLM（离线 Planner）
       ╱  ╲           几美元 / 分钟级 / 高泛化 / 低确定性
      ╱ <1%╲          职责：架构设计、策略生成、异常分析
     ╱──────╲
    ╱        ╲        L2  传统 ML（在线 Executor）
   ╱   ~5%    ╲       ≈ 0 成本 / 毫秒级 / 中泛化 / 统计确定性
  ╱────────────╲      职责：预测、分类、评分
 ╱              ╲
╱     ~10%       ╲    L3  确定性算法（决策引擎）
╱════════════════╲    0 成本 / 微秒级 / 零泛化 / 数学确定性
╱                  ╲  职责：定价公式、相似度计算、硬约束
╱       ~85%        ╲
╱════════════════════╲ L4  基础代码（数据管道）
                      0 成本 / 纳秒级 / 无智力 / 机械确定性
                      职责：清洗、正则提取、入库、过滤
```

**金字塔法则**：能用底层解决的绝不调上层。85% 的数据在 L4 被消化，95% 在 L4+L3 被过滤，只有不到 1% 的精英数据到达 L1。

---

## 一、四层定义

### L4：基础代码——金字塔基座

**硬件类比**：HDD（底层 / 海量 / 最便宜）

金字塔的基座，承接全部原始数据。纯机械执行，无任何智能，但吞吐量最高、成本为零。

| 组件 | 用途 |
|:---|:---|
| 正则表达式 | 邮箱电话提取、格式校验 |
| BeautifulSoup | HTML 标签过滤 |
| SQL | 结构化查询与入库 |
| Python 脚本 | 时区计算、数据类型转换 |

**输入**：原始网页（50,000 字）→ **输出**：清洗文本（2,000 字）+ 结构化字段

### L3：确定性算法——安全闸门

**硬件类比**：SSD（持久化 / 高吞吐）

金字塔的第二层，对 L4 输出做语义级过滤。数学公式驱动，零随机性，是财务安全的硬约束。

| 组件 | 用途 |
|:---|:---|
| TF-IDF / CountVectorizer | 文本向量化 |
| 余弦相似度 | 行业匹配度计算 |
| 运筹学公式 | 动态定价（分段函数 + 惩罚因子） |
| 轻量 Embedding | 跨语言语义对齐 |

**输入**：L4 清洗文本 → **输出**：相关度评分 + 是否进入 L2 的决策

### L2：传统 ML——核心推理

**硬件类比**：RAM（高速 / 容量适中）

金字塔的核心层，执行所有在线推理。本地 CPU 运行，成本趋近于零，输入一致则输出确定。

| 组件 | 用途 |
|:---|:---|
| XGBoost / LightGBM | 回归与分类 |
| LogisticRegression | 概率预测 |
| scikit-learn | 特征工程管道 |

**输入**：L3 过滤后的特征向量 → **输出**：预测值（回复率 / 库存需求 / 客户评分）

### L1：LLM——离线大脑

**硬件类比**：L1/L2 Cache（极快 / 极贵 / 极小）

金字塔的尖顶，不在生产路径上。只在离线周期（每周/每月）启动，输出的是**编译后的制品**——代码、公式、模型权重。

| 职责 | 输出制品 |
|:---|:---|
| 设计特征工程 | L2 训练脚本 |
| 分析异常长尾 | L3 熔断阈值调整 |
| 生成策略模板 | 开发信模板 |
| 重定价策略 | L3 定价公式参数 |

**成本**：几美元 Token 费。**关键约束**：L1 的输出必须固化到下层，不作为在线依赖。

## 二、向上卸载（Upward Offloading）

数据从金字塔底部向上流动，每一层过滤掉不需要的信息，只把精英数据送往上层。

### 路径一：流量卸载（L4/L3 消化 95%）

```
原始网页（50,000 字）
  ↓  L4：正则 + HTML 过滤
清洗文本（2,000 字）+ 邮箱电话
  ↓  L3：余弦相似度 > 0.05？
是 → 进入 L2        否 → 熔断拒绝
```

**错误做法**：原始网页直接丢给 LLM。**卸载后**：90% 的不相关商家在 L4/L3 被过滤，不惊动 L1/L2。

### 路径二：推理卸载（L2 替代 L1）

```
离线（L1）：LLM 设计特征工程 + 写训练代码
  ↓  交付制品
在线（L2）：LightGBM 本地预测成交概率
```

**错误做法**：每次客户判断都调 LLM Prompt。**卸载后**：在线 LLM 推理完全卸载为本地零成本矩阵运算。

### 路径三：决策卸载（L3 替代 L1）

```
离线（L1）：LLM 设计定价策略框架
  ↓  固化为公式
在线（L3）：分段函数 + 惩罚因子
  库存高 → 线性降价
  库存低 → 指数级提价
```

**错误做法**：让 LLM 理解供需关系并输出价格。**卸载后**：杜绝 AI 幻觉进入财务链路。

## 三、确定性梯度

金字塔从底到顶，确定性逐级递减：

```
L4  基础代码    ░░░░░░░░░░░░░░░░  机械确定性（逐字节确定）
L3  确定性算法  █░░░░░░░░░░░░░░░  数学确定性（公式固定）
L2  传统 ML     ████░░░░░░░░░░░░  统计确定性（输入一致 → 输出确定）
L1  LLM         ████████████░░░░  随机性（Temperature / Prompt 敏感）
```

B2B 商务和财务审计场景下，确定性是刚需。L1 的幻觉风险（$50 → $55）在 L3 的运筹学公式面前被彻底消除。

## 四、冷启动与多语言

### 冷启动：L4/L3 规则兜底

前一两个月没有足够样本训练 L2。L4/L3 层硬编码规则作为安全网：

```
IF 库存 > 需求 × 3 THEN 降价 10%
IF 客户 30 天未回复 THEN 切换跟进策略
```

规则引擎永远保留——随着数据积累，逐步平滑切换到 L2 ML 模型。

### 多语言：L3 轻量向量模型

Google Maps 商家官网遍布全球。L3 层引入本地跨语言向量：

| 模型 | 参数量 | 语言 | 部署 |
|:---|:---|:---|:---|
| MiniLM-L12 | 33M | 多语言 | 本地 CPU |
| FastText | 词向量 | 157 种 | 本地 CPU |
| LaBSE | 470M | 109 种 | 本地 CPU |

本地向量化 + 余弦相似度。不走 API，不消耗 Token，毫秒级响应。

## 五、系统容量

金字塔的形状决定了容量特征：基座无限扩展，尖顶成本固定。

```
数据量        L4/L3 消化     L2 推理       L1 Token 费
1,000 条     850            50            ~$0.01
10,000 条    8,500          500           ~$0.10
1,000,000 条 850,000        50,000        ~$10.00
```

L1 的 Token 成本与数据量**脱钩**——95% 的数据在底层被消化，永远不会到达 L1。

## 六、与 Agent 架构的关系

```
Agent 架构              智力金字塔
Planner (LLM)     →    L1 离线设计
Executor (Tool)    →    L2 在线执行
Memory            →    结构化数据库
Skill             →    L4 管道 + L3 规则 + L2 模型
```

Agent 的 Planner 和 Executor 运行时解耦但同处一次会话。金字塔模型中，L1 的输出是**编译后的制品**（模型权重、规则公式、特征管道），L2/L3/L4 独立运行——这是确定性的来源。

## 七、战略定位

```
卖数据     →  低壁垒，可复制
卖工具     →  中壁垒，需适配
卖解决方案  →  高壁垒，供需闭环本身是护城河
```

竞争对手复制的不是代码，是整条金字塔——L4 的清洗规则、L3 的定价公式、L2 的模型权重、L1 离线积累的策略知识。四级智力资产叠加，构成复合壁垒。
